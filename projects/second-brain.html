Second Brain

Aplikacja do zarządzania wiedzą i planowaniem z AI: ( RAG na dokumentach użytkownika- w przyszłości), notatki (TipTap), mapy myśli z importem i eksportem, zasoby z Firestore, kalendarz i zadania oraz panel Plan AI. Stack: Next.js 14 (App Router), Firebase (Auth, Firestore, Admin), Google Gemini. Hosting: Vercel.

// GITHUB REPO ↗

Problem → Rozwiązanie

Problem: Notatki, linki, dokumenty i plan działania często rozjeżdżają się między narzędziami; trudno w jednym miejscu łączyć własną bazę wiedzy z rozmową z modelem i realnym kalendarzem zadań bez spójnej warstwy danych i auth.

Rozwiązanie: Jedna aplikacja Next.js 14 (App Router) z Firebase (logowanie, Firestore, Admin po stronie API) i Gemini do RAG oraz asysty w planowaniu. Moduły Wiedza i Planowanie dzielą ten sam model użytkownika i persystencję w chmurze.

Kluczowe Funkcje

Wiedza — RAG i czat
Czat z własną bazą dokumentów użytkownika (embeddingi + odpowiedzi z kontekstem) w module Wiedza. (RAG na dokumentach użytkownika- w przyszłości)
Notatki TipTap
Edycja notatek bogatego tekstu z rozszerzeniami TipTap, spójna z resztą UI.
Mapy myśli
Drzewo poziome, notatki per węzeł, akcje AI (węzeł / poddrzewo), import tekstu lub obrazu przez API, eksport do HTML, zagnieżdżanie węzłów przeciągnięciem.
Zasoby i ulubione
Lista linków w Firestore z tagami i wyszukiwarką; pasek ulubionych z faviconami oraz akcje edycji i kopiowania URL.
Kalendarz i zadania
Widok kalendarza z segmentacją zdarzeń przez północ; zadania z terminem, statusem i notatką w modalu szczegółów; tagi i log aktywności.
Plan AI
Pasek rozmowy z modelem w kontekście planowania (historia wiadomości), zgodnie z architekturą modułu Planowanie w repozytorium.

Kluczowe Wyzwania Techniczne

  • RAG i koszty / jakość: Dobór fragmentów kontekstu, limity tokenów i spójność odpowiedzi Gemini względem dokumentów użytkownika.
  • Firebase + Next.js: Rozdział klienta (SDK web) i serwera (Admin), bezpieczne reguły Firestore oraz zmienne środowiskowe na Vercel.
  • Import map myśli: Obsługa tekstu i obrazu w API importu, stabilność formatu i UX edycji drzewa.
  • Kalendarz: Poprawne dzielenie wielodniowych zdarzeń na segmenty siatki godzin (expandEventToSegments).
  • Testy: Vitest (jednostkowe / integracyjne z emulatorem) oraz Playwright E2E — utrzymanie przy zmianach w API i UI.
  • Deploy: Konfiguracja domeny Vercel w Firebase Authorized domains oraz pełny zestaw sekretów zgodnie z .env.example.

Jak to działa?

1. Logowanie → Użytkownik uwierzytelnia się przez Firebase Auth; chronione trasy w grupie (protected).

2. Wiedza → Dokumenty i notatki trafiają do magazynu użytkownika; czat RAG woła API Next.js, które korzysta z Gemini i embeddingów.

3. Mapy i zasoby → Stan map myśli i kolekcja zasobów są zapisywane w Firestore; import i eksport przechodzą przez dedykowane endpointy API.

4. Planowanie → Zdarzenia kalendarza, zadania i log aktywności synchronizują się z backendem; Plan AI uzupełnia przepływ rozmową z modelem.

5. Admin i serwer → Operacje wymagające podniesionych uprawnień używają Firebase Admin w Route Handlers (app/api/).

6. Build i hosting → npm run build / Vercel; lokalnie dev na porcie 3100 (npm run dev).
