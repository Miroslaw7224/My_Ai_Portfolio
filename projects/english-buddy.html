English Buddy

Pełnoprawna aplikacja (PWA, responsywny UI) łącząca konwersacyjnego asystenta dopasowanego do poziomu, bazę fiszek z audio i harmonogram powtórek oraz narzędzia oceny umiejętności — z kontami użytkownika, opcjonalnym monitoringiem błędów i panelem QA dla administratorów.

Problem → Rozwiązanie

Problem: Uczący się angielskiego często mają rozproszone narzędzia: czat, słownictwo, powtórki i ocena poziomu dzieją się osobno; brakuje spójnego doświadczenia „tutor + ćwiczenia + feedback” zsynchronizowanego z CEFR i z progresją w czasie (SRS).

Rozwiązanie: Aplikacja Next.js 15 (App Router) z API routes, Supabase (auth + PostgreSQL z RLS) oraz OpenAI (czat, TTS, Realtime voice, agenty/oceny). Użytkownik wybiera lub weryfikuje poziom, uczy się przez moduły (czat, głos, fiszki, gramatyka itd.), a postęp i powtórki są zapisywane po stronie serwera zgodnie z konfiguracją profilu.

Kluczowe Funkcje

Poziomy CEFR (A1–C2)
Dopasowanie promptów, TTS i stylu korekty do poziomu; konfiguracja w kodzie (levelConfig / prompty).
Czat tekstowy i TTS
Rozmowa z AI z uwzględnieniem poziomu; synteza mowy (wiele głosów OpenAI).
Czat głosowy (Realtime)
Rozmowa w czasie rzeczywistym (WebRTC z fallbackiem do WebSocket — wg dokumentacji architektury).
Fiszki + SRS (SM-2)
Duży zestaw fiszek w data/flashcards, tryby quiz/guess, harmonogram dzienny i przyciski oceny trudności.
Test poziomujący i oceny
M.in. placement test (8 tur w README), feedback pisania i mówienia w wielu wymiarach (standardy typu IELTS/Cambridge w README).
Konto, demo i jakość
Supabase Auth, tryb demo z limitami, PWA i responsywność; panel QA dla adminów, Sentry (opcjonalnie).

Kluczowe Wyzwania Techniczne

  • Spójność AI z CEFR: Osobne prompty i zachowanie modelu per poziom, utrzymanie hierarchii promptów i spójności z ocenianiem.
  • Bezpieczeństwo danych użytkownika: PostgreSQL z RLS, operacje wrażliwe wyłącznie po stronie serwera (np. klucz serwisowy Supabase).
  • Realtime audio: Integracja OpenAI Realtime z WebRTC i obsługa fallbacku sieciowego.
  • Skala treści edukacyjnej: Tysiące fiszek w JSON, pipeline QA (walidacja schematu, smoke testy).
  • Tryb demo i limity: Ograniczenie nadużyć (np. liczba wiadomości/czas sesji) bez psucia UX.
  • Produkcja: Zmienne środowiskowe (OpenAI, Supabase, Sentry, cron secret), wdrożenie na Vercel, spójność publicznego URL (sitemap/robots vs dokumenty prawne).

Jak to działa?

1. Wejście → Logowanie przez Supabase lub start trybu demo (bez trwałego zapisu postępu).

2. Poziom CEFR → Wybór poziomu lub placement test weryfikujący poziom.

3. Moduły → Czat, czat głosowy, fiszki, gramatyka itd. z głównego pulpitu.

4. API → Frontend woła API Routes Next.js; serwer łączy się z OpenAI i/lub Supabase.

5. Stan i persystencja → Odpowiedzi i zdarzenia uczące aktualizują stan i cache (TanStack Query / Zustand), a trwałe dane (postęp, SRS) trafiają do bazy.

6. Powroty i admin → Harmonogram powtórek i statystyki; administrator może korzystać z panelu QA (przy uprawnieniach).
